Lisp has only two things?

∵ Mari-chan ∴ 2022-01-24 ∞ 3'

My baby functional programming

So here's a first interesting concept on Rasket: let, it's a way of defining a list of functions. The content of let dies after the lifetime of the define dies.

But how does lifetime work? As soon as there is no references/pointers to an object it's garbage. Talking about scope life times, same as C, let dies after the parenthesis that created it were close.

(define (checker p1 p2)
  (let ([p12 (hc-append p1 p2)]
        [p21 (hc-append p2 p1)])
    (vc-append p12 p21)))

(checker (colorize (square 10) "red")
           (colorize (square 10) "black"))

There are two things in a lisp language. Atoms and lists of symbolic expressions.

The way the lang. works is by using Eval, which eval things recursively: You evaluate a symbol by returning its value. Any other atom returns itself. If you quote something before whatever it will return itself. The way forms are evaluated is by recursevily evaluating what's inside procedures, and then to atoms:

```
(define x 10)
circle ( + 50 x)
```

This evaluate circle to a process(? forgot the name), + to a process, 50 to itself, x to the value in the scope.
